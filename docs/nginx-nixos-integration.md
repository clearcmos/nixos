# Nginx in NixOS vs. Debian: Key Differences

This document explains the key differences between configuring Nginx in NixOS compared to Debian, and covers important adaptations needed when migrating configurations.

## Configuration Philosophy Differences

### Debian Approach
- Files are placed in standard locations: `/etc/nginx/nginx.conf`
- Config can be directly edited by the administrator
- Service user and PID management handled through config files
- Dynamic module loading via `load_module` directive

### NixOS Approach
- Configuration is generated by NixOS based on the Nix expressions
- Files are placed in the Nix store and are immutable
- Service user and PID management handled by systemd/NixOS
- Modules are compiled into nginx during package build time

## Key Adaptation Points

When migrating from a Debian nginx configuration to NixOS, these are key areas that require adjustment:

### 1. User Directive

**Debian**: Uses a `user www-data;` directive in the config file.
```nginx
user www-data;
```

**NixOS**: The user is managed by systemd/NixOS automatically.
- Remove the user directive from your config, as it will generate warnings
- NixOS creates and manages appropriate service users

### 2. PID File Location

**Debian**: Often specified in the config with `pid /run/nginx.pid;`
```nginx
pid /run/nginx.pid;
```

**NixOS**: PID is managed by systemd and should be removed from config.
- Duplicate PID directives will cause nginx startup failures
- Error message will show: `"pid" directive is duplicate`

### 3. Module Loading

**Debian**: Dynamic modules loaded via `load_module` directive:
```nginx
load_module /usr/lib/nginx/modules/ngx_http_geoip2_module.so;
```

**NixOS**: Modules are compiled into the binary by overriding the package:
```nix
package = pkgs.nginx.override {
  modules = [ pkgs.nginxModules.geoip2 ];
};
```

### 4. File Paths

**Debian**: Standard file paths used:
```nginx
include /etc/nginx/mime.types;
geoip2 /usr/share/GeoIP/GeoLite2-Country.mmdb { ... }
```

**NixOS**: Paths reference the Nix store:
```nginx
include ${pkgs.nginx}/conf/mime.types;
geoip2 /var/lib/GeoIP/GeoLite2-Country.mmdb { ... }
```

## Implementation in Our System

Our NixOS configuration includes these adaptations to match the functionality of the Debian setup:

1. Removed the `user` directive to prevent warnings
2. Removed the `pid` directive to prevent startup failures 
3. Added the GeoIP2 module via package override instead of load_module
4. Adjusted paths to use NixOS-style locations
5. Added the nginx binary to system packages so it's available in PATH

## Troubleshooting Tips

If nginx fails to start:

1. Check logs with `cat /var/log/nginx/error.log` to identify issues
2. Look for duplicate directives that NixOS handles automatically
3. Verify your nginx configuration with `nginx -t`
4. Look for module path issues or incorrect includes

## Command Availability

In Debian, the `nginx` command is available directly in PATH. In NixOS, the command is available only if:

1. The nginx package is explicitly added to `environment.systemPackages`
2. You use an ephemeral shell with `nix-shell -p nginx`

Our configuration adds nginx to system packages so the command is directly available:
```nix
environment.systemPackages = with pkgs; [
  nginx
  certbot
];
```
